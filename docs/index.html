Usage
CLI Reference
claude-native [OPTIONS] [PATH]

Arguments:
  [PATH]              Project directory to scan [default: .]

Options:
  -f, --failures-only Only show failures and suggestions
  -o, --format        Output format [terminal, json]
      --init          Bootstrap CLAUDE.md, .claudeignore, settings.json
      --fix           Auto-apply quick-win suggestions
      --diff          Preview score improvement
      --watch         Live re-scoring on file changes
  -h, --help          Print help
  -V, --version       Print version

Exit codes:
  0  Score >= 40
  1  Score < 40 (CI integration: fail build)

CI Integration

# GitHub Actions
- name: Check Claude Native Score
  run: |
    cargo install claude-native
    claude-native -o json > score.json
    claude-native # exits 1 if <40

Philosophy
The 10 Commandments of Claude Native
  1. 01 Have a CLAUDE.md — concise, actionable, <200 lines
  2. 02 Ignore the noise — .claudeignore for deps, builds, generated files
  3. 03 Keep files small — <500 lines per file, <80 lines per function
  4. 04 Name things clearly — descriptive, consistent naming everywhere
  5. 05 Write tests — Claude's primary way to verify its own work
  6. 06 Be consistent — one pattern per concern, applied everywhere
  7. 07 Use types — contracts Claude can reason about
  8. 08 Structure logically — clear directories, obvious entry points
  9. 09 Automate guardrails — hooks for formatting, permissions for safe commands
  10. 10 Separate concerns — rules/ for guidance, skills for workflows, CLAUDE.md for essentials
